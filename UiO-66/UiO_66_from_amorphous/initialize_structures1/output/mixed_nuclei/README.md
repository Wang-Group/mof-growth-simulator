## Interpretation

The retained one-shot run shows the intended behavior:

- the initial mixed seed starts near `Zr6:Zr12 ~= 1:1`
- no new `Zr12` is added during growth
- `Zr12` is removed by repeated unlink + detached-fragment pruning events
- the structure continues to grow through `Zr6 + BDC` addition
- the retained final state is a single connected structure with `Zr12 = 0`

## Reproduction Note

This directory no longer stores the original mixed-seed files. To rerun the production case from scratch, first regenerate or restore the seed named `ratio_controlled_mixed_seed_160_seed612_frac055_fill6`, then run:

```powershell
python run_zr6_only_growth_case.py `
  --seed-pkls "output/mixed_nuclei/ratio_controlled_mixed_seed_160_seed612_frac055_fill6.pkl" `
  --replicates 4 `
  --base-rng-seed 2026040301 `
  --total-steps 100000 `
  --max-entities-delta 600 `
  --exchange-rxn-time-seconds 0.1 `
  --zr-conc 5000 `
  --linker-conc 69.1596872253079 `
  --capping-agent-conc 180 `
  --equilibrium-constant-coefficient 2.0 `
  --entropy-correction-coefficient 0.0 `
  --output-dir "output/mixed_nuclei/zr6_only_growth_runs" `
  --basename "oneshot_seed160_eq2_cap180_zr5000_pruned"
```

Regenerate the trajectory plot from the retained summary:

```powershell
python plot_zr6_only_growth_trace.py `
  --summary-json "output/mixed_nuclei/zr6_only_growth_runs/oneshot_seed160_eq2_cap180_zr5000_pruned/oneshot_seed160_eq2_cap180_zr5000_pruned.summary.json" `
  --snapshot-every-steps 1
```
